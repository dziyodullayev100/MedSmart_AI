MODUL 3: MA'LUMOTLAR BAZASI SXEMASI (DATABASE MODELS)
--------------------------------------------------------------------------------
Loyihada 5 ta asosiy model mavjud:

1. Doctor (Shifokor):
   - Fields: id(UUID), name, email, password(hashed), specialization, experience(int), phone, schedule(JSON).
2. Patient (Bemor):
   - Fields: id(UUID), name, email, password(hashed), phone, address, medicalHistory(Text).
3. Appointment (Uchrashuv):
   - Fields: id(UUID), patientId(FK), doctorId(FK), serviceId(FK), date(Date), time(String), status(Enum: scheduled, completed, cancelled), notes(Text).
4. Service (Xizmat):
   - Fields: id(UUID), name, description, price(Decimal), duration(String).
5. User (Tizim foydalanuvchisi - Admin/Staff):
   - Fields: id, username, password, role(Enum: admin, staff).

--------------------------------------------------------------------------------
MODUL 4: API SPECIFICATION (ROUTES)
--------------------------------------------------------------------------------
Base URL: http://localhost:5000/api

- GET /doctors      - Barcha shifokorlarni olish.
- POST /doctors     - Yangi shifokor qo'shish (Body required).
- GET /patients     - Bemorlar ro'yxati.
- POST /appointments - Uchrashuv belgilash (patientId, doctorId, date, time).
- PUT /appointments/:id/status - Holatni o'zgartirish (scheduled -> completed).
- GET /services     - Tibbiy xizmatlar ro'yxati.
- POST /users/login - Tizimga kirish (JWT token qaytaradi).
